# Security Policy

## Supported Versions

This project is a v0.1.0 public skill package. Security fixes are made on the
default branch until a stable release line exists.

## Reporting a Vulnerability

Please report suspected vulnerabilities through GitHub private vulnerability
reporting when available, or by opening a minimal public issue that avoids
sharing credentials, private repository data, unpublished launch material, or
third-party account details.

Include the affected version or commit, reproduction steps using synthetic
fixtures, and the expected impact. Do not include live access tokens, private
analytics, customer data, or embargoed launch plans in public reports.

## Data Handling

The skill is intended to produce local dry-run launch plans. It must not publish
packages, tag releases, post announcements, create GitHub releases, or mutate
external systems without a separate human-approved workflow.
